matchingContextdp back to ToC or Data Property ToC

IRI: https://w3id.org/won/matching#matchingContext

Specifies a 'matching context' for the Atom. Matchers should only send hints to this Atom if the counterpart specifies the same matching context. If no matching context is specified, matching is not restricted, and matchers may send hints with counterparts that do specify matching contexts.

NoHintForCounterpartni back to ToC or Named Individual ToC

IRI: https://w3id.org/won/matching#NoHintForCounterpart

An Atom specifying this flag indicates that matchers should not send Hint messages to the counterpart in case of a match. If the NoHintForMe flag is also present, matchers should disregard this atom completely.

UsedForTestingni back to ToC or Named Individual ToC

IRI: https://w3id.org/won/matching#UsedForTesting

Flag indicating the atom has been created for testing purposes. It should only be matched with other atoms flagged with UsedForTesting. The flag is intended to separate real-world atoms from those generated for testing the system, so that users are neither spammed nor misled.
Is defined by
https://w3id.org/won/core
